## 💎 企画要旨と要件定義

このモデルは、PVに依存せず、専門性の高いコンテンツとパーソナルデータを基に、**高単価なバックエンド商品（受注・コンサル）へのCVRを最大化**することを目的とします。

### 必須要件

1.  **専門コンテンツ（集客）:** Webメディアで「コアランゲージ、工学、未病」の専門コンテンツを提供し、信頼を獲得する。
2.  **無料会員登録:** コンテンツや診断アプリへのアクセス権をフックにリードを獲得する。
3.  **パーソナライズ診断:** アプリ/Webでユーザーのスキル、課題、体質などを深く特定し、データを取得する。
4.  **LLM個別育成:** 診断結果に基づき、LLMが**究極に個別最適化されたメルマガ**を生成・配信する。
5.  **デジアナ連携:** データに基づいた**物理的なパーソナルレポート**を郵送し、信頼性と権威性を確立する。
6.  **バックエンド受注:** 最終的に高単価なコンサルティング契約や案件受注に繋げる。

## 3\. 🛠️ 技術スタック

**「Next.js（フロント）」**、**「Headless WP（CMS）」**、\*\*「Django/Python（バックエンド/LLM連携）」\*\*の3層分離構造を採用し、高いパフォーマンスと拡張性を両立させます。

| レイヤー | 役割 | 主要技術要素 | ホスティング/連携 |
| :--- | :--- | :--- | :--- |
| **フロントエンド** | Webメディア、診断UIの高速表示、SEO | **Next.js** (Reactフレームワーク) | Vercel (高速ホスティング/SSR/ISR最適化) |
| **CMS** | 記事コンテンツの作成・管理 | **Headless WordPress** (REST APIでコンテンツ提供) | レンタルサーバー (管理の容易性) |
| **バックエンド/API** | ビジネスロジック、LLM連携、データ管理 | **Django / Python** (データ処理とAI連携に強み) | 専用クラウド (AWS, GCPなど) |
| **LLM・AI** | 個別メルマガ本文の生成 | **LLM API** (Gemini API など) | LLMプロンプト管理ロジックはDjango内で実行 |
| **データベース** | 全データの格納と分析 | **PostgreSQL** + **BigQuery** (DWHとして) | Djangoと連携 |
| **配信/デジアナ** | メール配信と物理発送 | **メール配信API** (SendGrid, Amazon SES)<br>**PDF生成ライブラリ** (郵送レポート用) | 外部API連携 |
| **決済/SFA** | 商品販売と顧客管理 | **Stripe** (決済)、**SFA/CRM** (Salesforce, Kintoneなど) | 外部SaaS連携 |
